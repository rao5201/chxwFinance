### 3.2 组件规范

#### 按钮样式
```css
/* 主按钮 */
.btn-primary {
  background: #E6AF2E;
  color: #0A192F;
  padding: 12px 24px;
  border-radius: 6px;

  transition: all 0.3s ease;

/* 次要按钮 */

  background: transparent;
  border: 1px solid #E6AF2E;
  color: #E6AF2E;
  padding: 12px 24px;
  border-radius: 6px;
}

/* 危险按钮 */
.btn-danger {
  background: #FF6B6B;
  color: white;
  padding: 12px 24px;
  border-radius: 6px;
}
```

```css

```


### 3.3 动画规范

#### 过渡动画
| 动画 | 时长 | 缓动函数 |
|------|------|----------|

| 透明度变化 | 200ms | ease-in-out |

- 卡片悬停: 上移 4px, 阴影增强
- 链接悬停: 颜色变为金色
- 加载动画: 旋转圆圈, 金色

#### HTML文件
- 前台页面: `index.html`, `market.html`, `trading.html`

- 使用小写字母，连字符分隔

#### CSS文件
- 主文件: `main.css`
- 组件: `components.css`
- 模块: `[module-name].css`
- 使用小写字母，连字符分隔

#### JavaScript文件
- 主文件: `main.js`
- 模块: `[module-name].js`
- 使用小写字母，连字符分隔

#### 图片文件
- 格式: `[category]-[name]-[size].[ext]`
- 示例: `banner-hero-1920x600.jpg`, `icon-user-64x64.png`
- 使用小写字母，连字符分隔

#### 文档文件
- 格式: `[文档名称].md`
- 使用中文名称，简洁明了

## 六、版本管理

### 6.1 版本号规范

采用语义化版本控制 (Semantic Versioning)

```
主版本号.次版本号.修订号
例如: v5.3.0
```

- **主版本号**: 重大更新，不兼容的API更改
- **次版本号**: 新功能，向后兼容
- **修订号**: 问题修复，向后兼容

### 6.2 版本发布流程

1. **开发阶段**
   - 功能开发
   - 单元测试
   - 代码审查

2. **测试阶段**
   - 集成测试
   - 性能测试
   - 安全测试

3. **发布阶段**
   - 版本打包
   - 更新日志
   - 正式发布

4. **维护阶段**
   - 监控运行
   - 收集反馈
   - 持续优化
